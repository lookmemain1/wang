## 1. 要做什么

做一个**纯前端、按脚本播放**的网页应用，用预置好的内容演示三个医院 AI 功能。它要看起来像一个可用的产品，但**不接后端、不接 API、没有真实 AI**。它在浏览器里离线运行，平板和笔记本都能跑，使用 Cognitas 品牌界面。临床内容全部由 Cognitas 提供；你负责做界面和脚本播放。

这是给医院决策者和世界银行看的概念演示。由我方人员按固定流程演示，不接触真实患者，不是试点系统。

## 2. 硬性要求

- **静态网页。** 不要后端、不要数据库、不要任何外部网络请求，所有资源打包在本地。
- **可离线运行。** 关掉 wifi 也能跑，从本地文件或本地静态服务器打开即可。
- **平板优先**（横屏，约 1024–1366px），笔记本也要正常显示。
- **内容与代码分离。** 所有临床内容从可编辑的 JSON 文件加载，非工程师也能改病例，不用动代码。
- **只用 Cognitas 品牌。** 界面上不得出现任何中文、供应商 logo 或第三方标识。
- **模拟流式输出（伪实时）。** 输出按定时器逐步显示，看起来像"实时生成"（每个功能的机制见下文）。
- **数据全部为合成数据。** 由 Cognitas 提供，无真实患者数据。

## 3. 整体界面

- 左侧栏（仅视觉，不需功能）：新建会话、历史记录、病历、患者、文件。
- 主区域：聊天式画布。
- 底部操作栏三个按钮，分别打开三个功能：**AI 写作（AI Writing）**、**医保助手（Insurance Assistant）**、**诊断助手（Diagnostic Assistant）**。
- 左上角放 Cognitas logo；整体为简洁的临床风格；界面文字用英文（可选英文 / 印尼文切换）。

请参照下面的交互流程（这些流程对标真实产品的体验，正是说服力所在），但全部换成 Cognitas 的视觉风格。

## 4. 三个功能

### 功能一 — Ambient AI（语音 → SOAP 结构化病历）

**入口：** 点 `AI Writing` → 下拉菜单 → `Outpatient Record`（门诊病历）。
**界面：** 全屏左右分栏。左 = "Live Transcription（实时转写）"，右 = "Outpatient Note（门诊病历）"实时生成。底部 = 动态声波 + 计时器 + 麦克风按钮 + 红色停止按钮。顶栏：返回 · Cancel · Confirm。

**播放机制（核心技巧）：**
1. 开始后，计时器和声波动画启动。
2. 左栏把转写稿逐段流式显示（每 ~0.5–1 秒出一段），看起来像实时语音转文字。逐字使用 **Transcript 文档 §3** 的转写稿。
3. 右栏比左栏延后几秒，分节填充 SOAP 病历，最终生成 **Transcript 文档 §4.1** 的完整结构化病历。展示体征自动填入（血压 148/92、BMI 27.6、HbA1c 7.4 等）。
4. 我方点红色停止 → `Confirm` → 病历存入"病历"列表（静态）。
5. 暂停/继续只需暂停定时器即可。

数字必须做归一化显示：口语 "7 koma 4" 显示成 `7.4`；"148 per 92" 显示成 `148/92`；"500 milligram tiga kali sehari" 显示成 `500 mg TID`。

### 功能二 — 医保编码（病历 → ICD-10 + ICD-9-CM）

**入口：** 点 `Insurance Assistant` → `Insurance Coding`。点输入框 → 弹出 "Patient Info" 弹窗 → 我方选一个预置病历（如 `Prolanis Note`、`TB Discharge`）→ `Start Analysis`。

**播放机制：**
1. 显示三步分析动画：*Parsing record（解析病历）→ Identifying key information（提取关键信息）→ Checking coding standards（核对编码标准）*（共约 3–5 秒）。
2. 渲染整页编码报告，包含：
   - 诊断对比表：**医生诊断** vs **AI 诊断 + 编码**。
   - 标注主诊断 / 次诊断。
   - 每个编码附一段来自病历原文的依据引用。
   - 相关的手术操作编码（ICD-9-CM）。

**内容来源：** Prolanis 病例 → **Templates §3**（主诊断 `E11.9`；次诊断 `I10`、`E78.2`、`E66.3`；若 UACR 确认则候选升级 `E11.21`）。TB 出院病例 → **Templates §5**（`A15.0` + `E11.9` + `D64.9`；操作编码 `87.42`、`87.44`、`89.07`）。

**关键：** 显示的编码必须是印尼 INA-CBG / BPJS 使用的标准 ICD-10（诊断）和 ICD-9-CM（操作）。**绝不能**出现任何外国系统的扩展子编码格式。

### 功能三 — CDSS / 药物相互作用警示

**入口：** 点 `Diagnostic Assistant` → `Risk Warning`。我方提交一份预填的患者信息 + 用药清单，或选一个已存患者。

**播放机制：**
1. 显示 "Searching medical databases（检索医学数据库）" 动画（数据库名称用中性、非中文的标识）。
2. 以文字流式输出结构化风险分析：严重程度结论 → 每组药物对的机制 → 处理建议（要点列表）→ 2–3 张文献引用卡片。

**内容来源：** 老年多重用药病例 → **Templates §6**（9 种药物，药物相关问题 DRP-1 至 DRP-7，硬警示：eGFR 48 / 72 岁用 Glibenclamide 违反 Beers 标准；ACE-I + NSAID + 利尿剂"triple whammy"；女性患者超适应症用 Tamsulosin；从 9 种药减到 4 种）。同时支持 Prolanis 病例的轻度警示（Captopril + Aspirin 提示级；Simvastatin + Amlodipine，Simvastatin 限 ≤20 mg）。

## 5. 演示病例（内容由 Cognitas 提供）

1. **Prolanis 2 型糖尿病 + 高血压（旗舰病例）** — 一个患者贯穿三个功能。转写稿和病历已就绪（见 Transcript 文档）。
2. **老年多重用药** — CDSS 专项展示（Templates §6）。
3. *（可选）* **TB 出院小结** — 多诊断编码 + 操作编码（Templates §5）。
4. *（可选）* **登革热住院** — 编码 `A91` + NSAID/阿司匹林硬性禁用拦截（Templates §4）。

先做病例 1 和 2，时间允许再加 3 和 4。

## 6. 分工

- **Cognitas 提供：** 全部转写稿、SOAP 病历、编码表、药物相互作用文案、病例 JSON，以及品牌素材（logo、配色）。
- **你负责：** 界面框架、导航、三个功能流程、脚本播放引擎、JSON 内容加载层、离线打包。

## 7. 技术建议

- 单页应用（React 或原生 JS），打包成静态文件，可从 `file://` 或本地静态服务器运行。
- 所有病例内容放在 `/content/*.json`（转写分段、病历分节、编码行、药物警示块），非工程师改文字不用动代码。
- 写一个简单的"脚本播放器"，读取时间轴数组，用 `setTimeout` 逐段显示。
- 交付为一个可拷到平板的文件夹；可选部署到 Vercel（设密码）供远程评审。

## 8. 不在本期范围

不做登录/鉴权，不做真实麦克风或语音识别，不做真实 AI，不做后端，不做 SatuSehat / FHIR 数据传输。源文档里的 FHIR 映射是后续路径二（白标集成）用的，本次忽略；如需视觉效果，可放一个静态的 "Sent to SatuSehat ✓" 角标。

## 9. 验收标准

- 我方能离线跑通完整的 Prolanis 流程：`AI Writing` → 病历实时生成 → `Confirm`；`Insurance Coding` → 报告渲染；`Risk Warning` → 警示流式输出。再加老年 CDSS 病例。
- 任何界面上都没有非 Cognitas 的品牌标识。
- 所有编码均为印尼标准（ICD-10 / ICD-9-CM）。
- 关掉 wifi 也能全程运行。
